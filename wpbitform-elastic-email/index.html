Bit Form Integration With ElasticEmail

Another integration that eases your email marketing is ElasticEmail integration via Bit form. When users submit any form, the email of that corresponding user will store in the newsletter list of ElasticEmail. For more details go through this documentation carefully.

Authorization Process‚Äã

  1. At first, select the form you want to apply the ElasticEmail integration. If you don‚Äôt have any then create a new one from Create Form. After that, follow the below instructions. Follow the path:

    Settings => Integrations => Click on (+) => Click on ‚Äòelastic-email‚Äô.

  2. Integration Name: Set the proper integration name. Default it's taken ElasticEmail.

Integration Settings‚Äã

  1. After that, you have to copy the API key from your Elastic Email account & paste the key on the field.
    To get the API key,

  • click on Elastic Email API Console or Login your Elastic Email dashboard

  • click on My Account => Setting => Create Additional API Key

  • You need to grant Full Access permission to make the API successful.

  • Now copy the API key and paste it into the integrations dashboard. And Click on Authorize button

  1. Click on Next to set up the other integration settings ahead.

  2. After completing the authorization process, Select the List from the drop-down option. You have to create a list before on your Elastic Email account. If you don't see any List. Click on the refresh button.

  3. You will find the Map Field option. You have to map all the necessary form fields with the related Elastic Email field. Here Email field is the default for Elastic Email. Now, click on the Next button to move forward for further settings.

  4. After clicking Finish & Save you can see the integration you have created now.

  5. After completing the settings, you have to set the conditions when this integration works. Don‚Äôt forget to specify the integration name; otherwise, the integration will not work correctly. Here we have set the condition that ‚Äì every time when records will be created/ edited, on form submission the integration will be triggered.

  6. Now, whenever a user will submit the form it will be added to the corresponding Campaign list of ElasticEmail. Also can view from form responses of Bit Form.

  7. You can view every success API of the submitted data from the timeline of related info.
